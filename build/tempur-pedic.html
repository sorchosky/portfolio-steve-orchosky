About the Project

Designing a new service to deploy landing pages across Tempur-Pedic’s retail network

This project demonstrates how I evolved a landing page service that my employer Icon Marketing provided to mattress retailers on behalf of Tempur-Pedic. For brand overview landing pages, Icon would design and deliver PDF and Photoshop artwork of customized templates for retailers to implement. While this worked for larger mattress retailers, small and mid-sized retailers that did not have a relationship with a website developer had difficulties deploying artwork into a functioning webpage. My solution was to develop these landing pages in HTML & CSS in-house and create a scalable plan for deployment across Tempur-Pedic’s retail network.

Background

Making it easier for consumers to shop online for one of America’s favorite mattresses

Tempur-Pedic was founded in 1998 based on their novel approach to adapting foam designed for space into mattresses. Their constant approach to innovation has won their products many accolades, including multiple J.D. Power awards for customer satisfaction and an induction into the Space Technology Hall of Fame. With them along the way for over 20 years has been Icon Marketing, a design and strategy agency based out of Greater Cincinnati.

While Icon also takes on national-level campaigns, the core relationship between them and Tempur-Pedic is to support Tempur-Pedic’s retail network. One component of this was to create landing pages.

Design

Making it easier for consumers to shop online for one of America’s favorite mattresses

Conveniently, this development lined up very closely with a new rebrand that Tempur-Pedic was undergoing. We used this to our advantage to update their landing page catalog to match the new look.

Development

Some retailer websites collected a lot of cobwebs over the years

When developing landing pages, my teammates and I noticed a lot of issues with code not translating properly when being implemented onto retailer websites. When looking for ways to troubleshoot, we found that retailers were wary about giving us access to access their websites via FTP. This led to a lot of trial-and-error when handing off developed files, and utilizing dev tools to target specific classes that needed to be updated.

After a few iterations of hacking fixes for retailers, I realized that a core issue was due to different front-end frameworks that retailers use causing overriding style code. Additionally, my knowledge of front-end development had grown tremendously, so I started modeling ways to optimize the code. This led me to converting our HTML & CSS into a full build system utilizing gulp to automate a lot of repetitive coding tasks. Some of these included auto-prefixing CSS parameters for legacy browsers, hyper-specifying CSS through using SCSS to wrap styles within a dedicated ID, and dividing SCSS files into design system-inspired modules that could be added or removed for optimization.

While these improvements optimized the pages in a lot of ways, some retailer websites still had technical concerns we couldn’t solve for. This led our team to develop two new landing page designs that used no JavaScript and very simplified HTML & CSS as an alternative.
